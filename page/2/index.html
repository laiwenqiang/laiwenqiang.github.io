Netty笔记01_boss和worker创初始化

发表于 2018-12-26 | 分类于 Netty

boss和worker的初始化，其实就是EventLoopGroup的初始化过程。

boss 和 worker

当Netty服务端启动，就会创建boss和worker，他们都是EventLoopGroup，只是负责的事情不同。

执行任务的时候都是调用EventLoopGroup中的EventLoop去执行。

boss通常只需要一个EventLoopGroup即可，用于处理客户端的连接请求。当有请求时，创建一个Channel，并且从EventLoopGroup(worker)中获取一个EventLoop，调用EventLoop的register方法，将Channel交给它处理。

worker包括多个EventLoopGroup，用于处理Channel上所有的读写事件。一个Channel只会被一个EventLoop处理，而一个EventLoop可能会被分配给多个Channel。

当然，有时候boss和worker可以是同一个。

boss和worker都会调用regiter方法，只是boss注册的是Selectionkey.OP_ACCEPT，而worker是Selectionkey.OP_READ。

一个EventLoopGroup包括一个或者多个EventLoop。
