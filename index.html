读书笔记02-Java并发编程实战

发表于 2017-11-04 | 分类于 读书笔记

第二部分

  1. 在正常的负载情况下，服务器应用程序应该同时表现出良好的吞吐量和快速的响应。而且，当负荷过载时，性能应该是逐渐降低，而不是直接失败。

  2. 大多数服务器应用程序都提供了一种自然的任务边界选择方式：以独立的客户请求为边界。

  3. Executor执行的任务由4个什么周期：创建，提交，开始和完成。

  4. 许多任务实际上都是存在延迟计算，执行数据库查询，从网络上获取资源，或者计算某个复杂的功能。对于这些任务，Callable是一种更好的抽象：它认为主入口（即call）将返回一个值，并且有可能抛出一个异常。

  5. Future表示一个任务的生命周期，并且提供了响应的方法来判断是否已经完成或者取消，以及获取任务的结果和取消任务等。

  6. 在java中没有一种安全的抢占方式来停止线程，因此也就没有安全的抢占方式来停止任务。只要一些协作的机制，使请求取消的任务和代码都遵循一种协商好的标志。

    其中一种协作机制能够设置某个“已请求取消”标志，而任务将定期地查看该标志。如果设置了这个标志，那么任务将会提前结束。

读书笔记01-Java并发编程实战

发表于 2017-11-02 | 分类于 读书笔记

第一部分

  1. 由于锁能够使其保护的代码路径以串行的方式来访问，因此可以通过锁来构造一些协议以实现对共享状态的独占访问。

  2. 同步方法可以确保单个操作的原子性，但是如果要把多个操作合并为一个复合操作时，需要额外的加锁机制。

  3. 当执行时间较长的计算时（网络IO或者控制台IO），不要持有锁。

  4. 对于同一个功能的代码，用相同的同步机制（同一个锁）来控制并发，不然容易造成混乱。

  5. 同步的两点：原子性和内存可见性。

    1. 某个线程在使用对象状态时，其他线程不能够同时修改状态。
    2. 某个线程修改了对象状态后，其他线程能够看到发生的变化。

  6. 重排性：在没有同步的情况下，代码的执行顺序可能出现一些调整。

  7. 最低安全性：在没有同步的情况下读取变量，得到的值可能已经过期。但是这个值是由之前某个线程设置的，而非一个随机的值。（java内存模型）

  8. 对于非volatile类型的64位数值变量（double和long），最低安全性不生效。

    java内存模型要求，变量的读取和写入操作都必须是原子性。但是对于double和long，jvm允许将64位的读/写分为两个32位进行操作。

JDK动态代理

发表于 2017-08-15 | 分类于 java基础

静态代理是由被代理类（A）和代理类（B），同时实现接口（I），并且B拥有A的引用。
而动态代理的作用是利用反射原理，让程序自动生成代理类（B），减少对具有相同功能的代理类的编写。如日志功能。

UML类图

JDK的动态代理产生的代理类（$Proxy01），也会实现接口（Subject），然而他不会直接引用被代理类（RealSubject），而是会由它的父类（Proxy）里的InvocationHandler的实现类（MyInvocationHandler）引用。
