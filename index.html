JUC-AQS理解（一）

发表于 2017-07-07 | 分类于 Java基础

AQS（AbstractQueuedSynchronizer）类，是java.util.corrent包的核心部分。它用模板模式构建起来，主要是维护了一个CLH队列（在上章节有提到过），用来调度队列中的线程挂起和唤醒等操作。

acquire方法

接上章节，ReentrantLock会调用acquire(1)来进行线程的挂起和唤醒操作。acquire(1)作为AQS的一个重要入口，代码如下：

  1. tryAcquire()是一个尝试获取资源锁的方法，由子类去实现的。例如：ReentrantLock的公平锁和非公平锁，他们根据自己的需求去实现它。
  2. addWaiter()会创建一个存放当前线程的节点，并且将该节点放入等待队列之中。
  3. acquireQueued()当前线程放入队列后，检查是否需要挂起。
  4. selfInterrupt()比较好理解，就是将当前线程挂起。

子类调用父类的synchronized方法

发表于 2017-06-27 | 分类于 Java基础

本文是对于Stack的push方法思考而得来。当子类调用父类的synchronized方法时，获取的同步锁是子类的，而非父类的。

Stack类的方法

java.util.Stack类的方法不多，简要说一下。

由synchronized修饰的有：

  1. public synchronized E pop()
  2. public synchronized E peek()
  3. public synchronized int search(Object o)

没有synchronized修饰的有：

  1. public E push(E item)
  2. public boolean empty()

empty()方法的作用是，判断存储的数据量是否为0。
