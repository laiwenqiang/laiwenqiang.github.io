Netty笔记_boss和worker创建

发表于 2018-12-26 | 分类于 Netty

boss和worker的初始化，其实就是EventLoopGroup的初始化过程。

boss 和 worker

当Netty服务端启动，就会创建boss和worker，他们都是EventLoopGroup，只是负责的事情不同。

执行任务的时候都是调用EventLoopGroup中的EventLoop去执行。

boss通常只需要一个EventLoop即可，用于处理客户端的连接请求。当有请求时，创建一个Channel，并且从EventLoopGroup(worker)中获取一个EventLoop，调用EventLoop的register方法，将Channel交给它处理。

worker包括多个EventLoop，用于处理Channel上所有的读写事件。一个Channel只会被一个EventLoop处理，而一个EventLoop可能会被分配给多个Channel。

当然，有时候boss和worker可以是同一个。

boss和worker都会调用regiter方法，只是boss注册的是Selectionkey.OP_ACCEPT，而worker是Selectionkey.OP_READ。

每个child(EventLoop)都会创建selector，boss用于监听请求，worker用于处理数据及其业务。

关于SelectorTuple
这个顾名思义，就是selector元组，还是不太懂，其实就是个Bean，只装了selector和unwrappedSelector。

selector：是Netty自己封装的类–SelectedSelectionKeySetSelector，继承于NIO的Selector。不过里面存放着NIO的Selector和SelectedSelectionKeySet（也是继承于NIO的SelectionKey），同时还重写了select()等方法。总的来说，是对NIO的Selector的代理。由此可见，这个这个类对于请求监听及数据处理的重要性。
unwrappedSelector：没有被封装的Selector，就是NIO里面的那个Selector，实现类是“sun.nio.ch.SelectorImpl”。
