我们先调用startJob()方法，启动EventLoop线程，执行task，马上打印”begin execute…”，然后睡眠5s，让出CPU资源。在此期间，主线程和EventLoop线程同时RUNNING，竞争CPU，所以”before sync”是有可能先打印的。future.sync();这行代码会调用await()方法阻塞，等待唤醒。5s后，”after execute…”被打印，task任务执行完成。EventLoop继续执行下一个setSuccess()任务，将promise设置为success。在promise.setSuccess(null)中，会唤醒之前被阻塞的主线程，得以打印”after sync”。然后，该方法会把之前addListener()注册进来的任务进行执行，打印”listener completed! Success: true”。

关于initAndRegister

1）如果initAndRegister()方法出现异常或者其他校验不通过的情况，在该方法中将会调用promise.setFailure(…)方法，然后返回。

2）由initAndRegister() -> AbstractChannel$AbstractUnsafe#register()时，会开启EventLoop的线程进行注册，使用在主线程中才有接下来的register.isDone()判断，已经天剑listener等操作。

3）在注册时，如果注册成功，在AbstractChannel$AbstractUnsafe会调用register() -> register0() -> safeSetSuccess()，这里promise.trySuccess()方法将会进行设置成功；注册失败，则会调用promise.tryFailure(cause)进行设置。

注册完成后

如果注册成功，在register0()方法在将会运行promise.trySucces()这行代码。(这里的promise也就是regFuture) 如果注册失败，在register0()方法中将会运行promise.tryFailure(cause)这行代码。(同上)

如果register.isDone()为真，表示已经完成了注册任务（此时，可能注册成功或者失败）。那么将会进入doBind0()方法进行注册绑定任务。这个任务的Future相关部分，也是一样的。

如果register.isDone()为假，次需要添加一个listener监听，等待其完成注册任务后回调这个监听任务，从而进入doBind0()这个方法。

最后

最后再看看一开始的boot.bind().sync()这段代码，虽然主线程阻塞在sync()这里，但是eventLoop线程最终会调用promise的setSuccess()或者setFailure()，由此去唤醒主线程，使其得以继续往下执行。

同理，主线程的future.channel().closeFuture().sync()这段代码也是一样，closeFuture()返回一个future，然后阻塞在sync()，等待其他线程调用这个future的setSuccess()或者setFailure()，由此唤醒。其他的也是类似的原理。
